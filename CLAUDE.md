## Project Overview

Wordle solver web app. Users enter guesses with color-coded results (green/yellow/red), and the solver suggests remaining candidates ranked by elimination power and word frequency.

## Development Commands

### Backend (FastAPI)
```bash
# Install dependencies
pip install -r requirements.txt

# Run dev server (port 8000)
uvicorn api:app --reload
```

### Frontend (Vue 3 + Vite)
```bash
cd frontend
pnpm install
pnpm dev      # Dev server on port 5173, proxies /api to localhost:8000
pnpm build    # Production build to dist/
```

Both servers must run simultaneously for development. The Vite dev server proxies `/api/*` requests to the FastAPI backend (configured in `frontend/vite.config.js`).

## Architecture

**Backend (`api.py`):** FastAPI app with the core solver logic:
- Words encoded as 2028-dimensional binary vectors (green/yellow/red × 26 letters × 5 positions)
- Constraints from guesses accumulated via bitwise OR on vectors
- Candidate filtering via vectorized NumPy matrix multiplication
- Elimination scoring simulates all possible response patterns per candidate word; more distinct patterns = more informative guess
- Only top ~80 candidates are scored for elimination to keep response time fast

**API endpoints:** `GET /api/starting-words`, `GET /api/words`, `POST /api/solve`, `GET /api/suggest/{word}`

**Frontend (`frontend/src/`):** Vue 3 with Composition API, no state library:
- `App.vue` — main state management, API calls to `/api/solve`
- `GuessRow.vue` — guess input with clickable color-cycling tiles (red→yellow→green)
- `WordCloud.vue` — candidates sized by elimination score, colored by rank
- `ConstraintInfo.vue` — displays confirmed positions, required letters, candidate count

## Data Files

- `twl06.txt` — TWL06 Scrabble dictionary (source of valid 5-letter words)
- `unigram_freq.csv` — word frequency data for ranking candidates
- `words.json` — pre-computed response pattern matrix (~731MB, generated by `wordle_ml.py` on first run)

## Supporting Scripts

- `wordle_ml.py` — CLI solver and `words.json` generator (input format: `word result` where result is g/y/r per letter)
- `wordle.py` — game simulator for testing solver against chosen/random words
- `word_guess.py` — interactive regex-based word finder
- `most_matches.py` — analysis of words with maximum pattern diversity
